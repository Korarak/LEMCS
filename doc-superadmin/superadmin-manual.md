## บทที่ 1: บทนำและภาพรวมระบบ

### 1.1 ระบบ LEMCS คืออะไร

LEMCS (Loei Educational MindCare System) เป็นระบบประเมินสุขภาพจิตนักเรียนระดับดิจิทัล ครอบคลุมนักเรียนระดับอนุบาล–มัธยมปลาย ในจังหวัดเลยกว่า **100,000 คน** รองรับ 4 สังกัดหลัก ได้แก่ สพฐ., สพม., สอศ. (สอศ./อาชีวะ), สช. (เอกชน) และ สกร.

แบบประเมินที่ใช้ในระบบ:

| แบบประเมิน | ชื่อเต็ม | กลุ่มอายุ | คะแนนเต็ม |
|---|---|---|---|
| **ST-5** | แบบประเมินความเครียด | อายุ ≥ 15 ปี | 0–15 |
| **PHQ-A** | Patient Health Questionnaire-Adolescent | อายุ 11–20 ปี | 0–27 |
| **CDI** | Children's Depression Inventory | อายุ 7–17 ปี | 0–54 |

### 1.2 บทบาท Superadmin

Superadmin คือผู้ดูแลระบบระดับจังหวัด (สำนักงานศึกษาธิการจังหวัดเลย) มีสิทธิ์ดูข้อมูลทุกสังกัด ทุกอำเภอ และทุกโรงเรียนโดยไม่ถูกจำกัดขอบเขต

**สิทธิ์ที่ Superadmin มี:**

| หมวดงาน | ดูข้อมูล | แก้ไข/ดำเนินการ |
|---|:---:|:---:|
| Dashboard & Analytics | ✅ | ✅ (Export/Download) |
| รอบสำรวจ | ✅ | ✅ (เปิด/ปิด/ยกเลิก/ลบ) |
| ระบบแจ้งเตือน (Alerts) | ✅ | ✅ (อัปเดตสถานะ) |
| รายงานรายบุคคล | ✅ | ✅ (Export) |
| ข้อมูลนักเรียน | ✅ | ✅ (แก้ไขข้อมูลพื้นฐาน/NID) |
| ผู้ใช้งานระบบ | ✅ | ✅ (สร้าง/แก้ไข/Reset password) |
| ข้อมูลโรงเรียน | ✅ | ❌ (เฉพาะ systemadmin) |
| Audit Logs | ✅ | ❌ (ดูอย่างเดียว) |
| นำเข้าข้อมูล (Import) | ❌ | ❌ (เฉพาะ systemadmin) |
| โครงสร้างองค์กร | ❌ | ❌ (เฉพาะ systemadmin) |

### 1.3 โครงสร้างลำดับชั้นของระบบ

```
systemadmin  (ทีมพัฒนา — สิทธิ์สูงสุด)
    └── superadmin  (ศึกษาธิการจังหวัด)
            └── commissionadmin  (ศึกษาธิการอำเภอ / สังกัด)
                    └── schooladmin  (ผู้บริหารโรงเรียน)
                            └── student  (นักเรียน)
```

### 3.1 FilterBar — การกรองข้อมูล

FilterBar อยู่ด้านบนสุดของ Dashboard ใช้ควบคุมข้อมูลที่แสดงใน **ทุก Chart และทุก Section พร้อมกัน**

มีพารามิเตอร์ทั้งหมด **9 ตัว:**

| พารามิเตอร์ | คำอธิบาย |
|---|---|
| รอบสำรวจ | เลือกดูเฉพาะรอบที่ต้องการ |
| สังกัด (Affiliation) | สพฐ. / สพม. / สอศ. / สกร. / สช. |
| อำเภอ (District) | อัปเดตอัตโนมัติตามสังกัดที่เลือก |
| โรงเรียน (School) | อัปเดตอัตโนมัติตามอำเภอที่เลือก |
| ประเภทแบบประเมิน | ST-5 / PHQ-A 0/ CDI |
| ชั้นเรียน (Grade) | ป.1–ป.6, ม.1–ม.6, ปวช./ปวส. |
| เพศ (Gender) | ชาย / หญิง |
| วันที่เริ่ม | วันที่เริ่มต้นที่ต้องการดู |
| วันที่สิ้นสุด | วันที่สิ้นสุดที่ต้องการดู |

> **เคล็ดลับ:** DropDown สังกัด → อำเภอ → โรงเรียน เชื่อมกันเป็น Hierarchical cascade เลือกสังกัดแล้วอำเภอจะอัปเดตเองอัตโนมัติ

---

> 📸 **[แคปภาพ: FilterBar — แสดง Dropdown ทั้ง 9 ตัวและปุ่ม Apply/Reset]**

### 3.2 Section A — สรุปภาพรวมผู้บริหาร

**Refresh อัตโนมัติทุก 60 วินาที**

#### ① StatsCards — ตัวเลขหลัก 4 ตัว

แสดงตัวเลขสำคัญที่มองเห็นได้ทันที:

- **นักเรียนทั้งหมด** — จำนวนนักเรียนในระบบ ณ ปัจจุบัน
- **ทำแล้วในรอบนี้** — จำนวน + % coverage ของรอบที่เปิดอยู่
- **ความเสี่ยง moderate+** — นักเรียนที่ต้องติดตามโดยด่วน
- **Alert รอดำเนินการ** — เคสที่ยังไม่มีผู้รับไปดูแล

---

> 📸 **[แคปภาพ: StatsCards ทั้ง 4 การ์ด]**

---

#### ② RiskFunnelChart — Funnel แสดงเส้นทางการดูแล

แผนภูมิ Funnel 4 ขั้นแสดงจำนวนนักเรียนที่ลดหลั่นในแต่ละขั้นตอน:

```
ลงทะเบียน  →  ทำแบบประเมินแล้ว  →  พบความเสี่ยง  →  ดูแลแล้ว
 100,000          45,000               1,200            980
```

ใช้ระบุว่ากระบวนการดูแลมี **ช่องว่างตรงไหน** เช่น นักเรียนทำแบบประเมินน้อยหรือเคสเสี่ยงยังไม่ได้รับการดูแล

---

> 📸 **[แคปภาพ: RiskFunnelChart — แสดง 4 ขั้น พร้อมตัวเลขในแต่ละขั้น]**

#### ③ InsightPanel — ข้อสรุปอัตโนมัติ

### 3.4 Section C — วิเคราะห์เชิงลึก (Analytics)

5 กราฟวิเคราะห์ข้อมูลเชิงลึก:

#### 1) TrendChart — แนวโน้ม 6 เดือนล่าสุด

- **แกน X:** เดือน (ต.ค.–มี.ค. หรือตามช่วงที่เลือก)
- **แกน Y:** จำนวนที่พบความเสี่ยง
- 3 เส้นแยกสี: 🟡 ST-5 · 🔵 PHQ-A · 🟢 CDI
- ดูได้ทันทีว่าช่วงไหนมีความเสี่ยงพุ่งสูง (เช่น ช่วงสอบ)

---

> 📸 **[แคปภาพ: TrendChart — Line Chart แสดง 3 เส้นของ ST-5/PHQ-A/CDI]**

---

#### 2) MoMDeltaChart — เปรียบเทียบเดือนนี้ vs เดือนก่อน

- Bar 🔴 แดง = ความเสี่ยงเพิ่มขึ้น (น่าเป็นห่วง)
- Bar 🟢 เขียว = ความเสี่ยงลดลง (ดีขึ้น)
- หน่วยเป็น % เช่น PHQ-A +12% หรือ CDI −5%

#### 3) SeverityChart — Doughnut สัดส่วนระดับความเสี่ยง

| สี | ระดับ | การดำเนินการ |
|---|---|---|
| 🟢 เขียว | ปกติ | ไม่ต้องดำเนินการเพิ่ม |
| 🟡 เหลือง | เล็กน้อย | ให้คำแนะนำ |
| 🟠 ส้ม | ปานกลาง | ครูแนะแนวดูแล |
| 🔴 แดง | รุนแรง | ต้องดำเนินการทันที |

ตัวเลขตรงกลาง = จำนวนนักเรียนที่ประเมินแล้วทั้งหมด

#### 4) AssessmentTypeChart — Stacked Bar แยกประเภทแบบประเมิน

- แกน X: ST-5 / PHQ-A / CDI
- แต่ละ Bar = สัดส่วน % ทุกระดับรวม 100%
- เห็นได้ทันทีว่าแบบประเมินไหนพบความเสี่ยงรุนแรงมากที่สุด

#### 5) RiskProgressBars — แถบ Progress 4 ระดับ

- แสดง N คน (xx%) ในแต่ละระดับ
- กดที่แถบเพื่อ **drill-down ดูรายชื่อนักเรียน** ในระดับนั้น

> 📸 **[แคปภาพ: Section C — แสดงกราฟทั้ง 5 ตัว]**

### 3.5 Section D — เปรียบเทียบองค์กร (Comparison)

#### OrgCompareChart — 100% Stacked Bar

- ทุก Bar ยาว **100%** เสมอ เพื่อเปรียบเทียบ **สัดส่วน %** ได้ยุติธรรม ไม่ถูกบิดเบือนโดยขนาดโรงเรียน
- โรงเรียนเล็ก 300 คน และโรงเรียนใหญ่ 3,000 คน สามารถเปรียบเทียบกันได้
- สีแต่ละส่วน = ระดับความเสี่ยง (เขียว/เหลือง/ส้ม/แดง)
- มองเห็นทันทีว่าองค์กรใดมี % สีแดง+ส้มมากกว่ากัน → ต้องให้ทรัพยากรเพิ่ม

**Auto-grouping — ระบบจัดกลุ่มอัตโนมัติตาม FilterBar:**

| FilterBar ที่เลือก | ระบบจัดกลุ่มโดย | ตัวอย่าง |

| ไม่กรอง (ทั้งจังหวัด) | **สังกัด** | สพฐ. / สอศ. / สช. / สกร. (4 bars) |
| เลือก Affiliation | **อำเภอ** | 14 อำเภอในจังหวัดเลย |
| เลือก District | **โรงเรียน** | ทุกโรงเรียนในอำเภอที่เลือก |

> 📸 **[แคปภาพ: Section D — OrgCompareChart แสดง 100% Stacked Bar เปรียบเทียบสังกัด/อำเภอ/โรงเรียน]**

## บทที่ 4: การจัดการรอบสำรวจ

**เส้นทาง:** `/admin/survey-rounds`

รอบสำรวจ (Survey Round) คือช่วงเวลาที่นักเรียนสามารถเข้าทำแบบประเมินได้ **ต้องเปิดรอบก่อนนักเรียนถึงจะทำแบบประเมินได้**

---

> 📸 **[แคปภาพ: หน้า Survey Rounds — แสดงตารางรอบสำรวจทั้งหมดพร้อมสถานะ]**

---

### 4.1 สถานะของรอบสำรวจ

| สถานะ | สัญลักษณ์ | ความหมาย |
|---|---|---|
| เปิด (Open) | 🟢 กะพริบ | นักเรียนทำแบบประเมินได้ |
| ปิด (Closed) | ⚫ | สิ้นสุดการเก็บข้อมูล รวมในรายงาน |
| ยกเลิก (Cancelled) | ⏸ | หยุดชั่วคราว ข้อมูลไม่นับในรายงาน |

### 4.2 การเปิดรอบสำรวจใหม่

1. คลิกปุ่ม **"➕ เปิดรอบใหม่"**
2. กรอกข้อมูลในฟอร์ม:
   - **ชื่อรอบ (Label):** เช่น `ภาคเรียน 1/2568` หรือ `มิถุนายน 2568`
   - **ปีการศึกษา (Academic Year):** เลือกปี พ.ศ.
   - **ภาคเรียน (Term):** 1 หรือ 2
3. คลิก **"เปิดรอบ"** → ระบบสร้าง SurveyRound สถานะ "open"
4. นักเรียนจะเห็น **Banner สีเขียว** บน Dashboard ของตนเอง

> ⚠️ **ข้อควรระวัง:** ไม่ควรเปิดหลายรอบพร้อมกัน เพราะจะทำให้นักเรียนสับสน

---

> 📸 **[แคปภาพ: Modal เปิดรอบสำรวจใหม่ — แสดงช่องกรอก Label/ปีการศึกษา/ภาคเรียน]**

### 4.3 การปิดรอบสำรวจ

1. คลิกปุ่ม **"ปิดรอบ"** ที่แถวของรอบที่ต้องการ
2. ยืนยันการปิด
3. ผลที่ตามมา:
   - ✅ ข้อมูลทั้งหมดยังคงอยู่ในระบบ
   - ✅ รวมในสถิติและรายงาน
   - ❌ นักเรียนไม่สามารถส่งแบบประเมินใหม่ได้

### 4.4 การยกเลิกรอบสำรวจ (Cancel)

1. คลิกปุ่ม **"⏸ ยกเลิกรอบ"**
2. ยืนยันการยกเลิก
3. ผลที่ตามมา:
   - ✅ ข้อมูลทั้งหมดยังคงอยู่ในฐานข้อมูล (Audit Trail)
   - ✅ Alert ที่สร้างไว้ยังคงอยู่ (เพื่อความปลอดภัยของนักเรียน)
   - ❌ **ไม่นับในสถิติ** และรายงานทั่วไป
   - ❌ นักเรียนไม่สามารถส่งแบบประเมินใหม่ได้

> **ใช้ Cancel เมื่อ:** เปิดรอบผิดพลาด หรือต้องการระงับชั่วคราวโดยยังเก็บข้อมูลไว้

### 4.5 การลบรอบสำรวจ (⚠️ ข้อมูลจะหายถาวร)

1. คลิกปุ่ม **"🗑️ ลบรอบ"**
2. ระบบแสดง Modal ยืนยัน — ต้องพิมพ์ **"ลบถาวร"** เพื่อยืนยัน
3. ผลที่ตามมา:
   - ❌ ลบ **แบบประเมินทั้งหมด** ในรอบนั้น — **ไม่สามารถกู้คืนได้**
   - ✅ Alert ยังคงอยู่ (ข้อมูลความปลอดภัยไม่ถูกลบ)

> 🚨 **คำเตือน:** ใช้การลบเฉพาะกรณีที่รอบเปิดผิดพลาดและยังไม่มีนักเรียนทำแบบประเมิน หรือต้องการล้างข้อมูล Test เท่านั้น

## บทที่ 5: ระบบแจ้งเตือนความเสี่ยง

**เส้นทาง:** `/admin/alerts`

ระบบสร้าง Alert อัตโนมัติเมื่อนักเรียนมีผลการประเมินที่น่ากังวล

### 5.1 เงื่อนไขการสร้าง Alert

| แบบประเมิน | เงื่อนไข | ระดับ |
|---|---|---|
| PHQ-A | Q9 ≥ 1 คะแนน | 🚨 Critical (Suicide Risk) |
| PHQ-A | BQ1 หรือ BQ2 = true | 🚨 Critical (Suicide Risk) |
| CDI | คะแนนรวม ≥ 15 | 🔴 Urgent (Clinical) |
| ST-5 / PHQ-A / CDI | ความเสี่ยงระดับ moderate+ | ⚠️ Warning |

> **สำคัญ:** Alert ประเภท Suicide Risk จะแจ้งเตือนใน Dashboard **ทุกระดับพร้อมกัน** (ครู → ผู้บริหารอำเภอ → จังหวัด) โดยไม่ผ่านคิว (Synchronous)

### 5.2 กระบวนการจัดการ Alert

---

> 📸 **[แคปภาพ: หน้า Alerts — แสดงตารางรายการ Alert พร้อม Badge สถานะและระดับ]**

---

#### ขั้นตอนการดำเนินการ

```
ตรวจพบ (Alert สร้างอัตโนมัติ)
        ↓
รอดำเนินการ  →  กำลังดูแล  →  เสร็จสิ้น
                                    ↑
                              หรือ "Referred" (ส่งต่อผู้เชี่ยวชาญ)
```

### 5.3 การกรอง Alert

| ตัวกรอง | ตัวเลือก |
|---|---|
| สถานะ | new / acknowledged / in_progress / referred / closed |
| ระดับ | warning / urgent / critical |
| สังกัด → อำเภอ → โรงเรียน | Cascade dropdown |

| ช่วงวันที่ | วันเริ่ม–สิ้นสุด |

### 5.4 การอัปเดตสถานะ Alert

1. ค้นหา Alert ที่ต้องการจัดการ
2. คลิกปุ่ม **"เปลี่ยนสถานะ"** ในแถวนั้น
3. เลือกสถานะใหม่:
   - **Acknowledged** — รับทราบแล้ว
   - **In Progress** — กำลังดำเนินการ
   - **Referred** — ส่งต่อนักจิตวิทยา/ผู้เชี่ยวชาญ
   - **Closed** — ดำเนินการเสร็จสิ้น
4. บันทึกหมายเหตุ (ถ้ามี)

---

> 📸 **[แคปภาพ: Modal อัปเดตสถานะ Alert — แสดงตัวเลือกสถานะและช่องหมายเหตุ]**

แสดง 50 รายการต่อหน้า พร้อม Pagination

### 10.3 ความสำคัญทางกฎหมาย

การมี Audit Log แสดงให้เห็นว่าระบบมีความโปร่งใสในการเข้าถึงข้อมูลส่วนบุคคล ตามมาตรา 36 ของ พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562

> 📋 **แนะนำ:** ตรวจสอบ Audit Log เป็นประจำทุกเดือนเพื่อให้แน่ใจว่าการเข้าถึงข้อมูลเป็นไปตามที่ได้รับอนุญาต

         /admin/users → ยืนยันว่า schooladmin ทุกโรงเรียนบัญชีเปิดอยู่
         ↓
ขั้น 3: เปิดรอบสำรวจใหม่
         /admin/survey-rounds → กด "เปิดรอบใหม่"
         กรอก label เช่น "ภาคเรียน 1/2568"
         ↓
ขั้น 4: แจ้งโรงเรียน
         แจ้งให้ schooladmin แต่ละโรงเรียนทราบว่าเปิดรอบแล้ว
         ↓
ขั้น 5: ติดตาม Dashboard
         /admin/dashboard → ดูความคืบหน้าการทำแบบประเมิน (auto-refresh 60s)

## ภาคผนวก: คำถามที่พบบ่อย

**Q: นักเรียนบอกว่าทำแบบประเมินไม่ได้ ทั้งที่ Login ได้แล้ว**
> A: ตรวจสอบว่ามีรอบสำรวจที่เปิดอยู่หรือไม่ที่ `/admin/survey-rounds` ถ้าไม่มีรอบที่สถานะ "เปิด" นักเรียนจะเห็น Banner สีเหลือง "ยังไม่เปิดรอบการสำรวจ"

**Q: ลืมจะปิดรอบสำรวจ นักเรียนยังส่งข้อมูลมาเรื่อยๆ**
> A: ไปที่ `/admin/survey-rounds` → คลิก "ปิดรอบ" หรือ "⏸ ยกเลิกรอบ" ได้ทันที ข้อมูลที่ส่งมาแล้วจะไม่หายหาย

**Q: Alert ขึ้นเยอะมาก จะกรองเฉพาะที่สำคัญได้อย่างไร**
> A: ที่ `/admin/alerts` กรองสถานะ = "new" และระดับ = "critical" จะเห็นเฉพาะเคสใหม่ที่ยังไม่มีผู้รับและมีความเสี่ยงสูงสุด

**Q: ส่งออก Excel แล้วไม่มีข้อมูล**
> A: ตรวจสอบว่า FilterBar ไม่ได้กรองข้อมูลจนเหลือ 0 รายการ ลองกด "ล้างตัวกรอง" แล้ว Export ใหม่

**Q: โรงเรียนใหม่ยังไม่อยู่ในระบบ จะทำอย่างไร**
> A: Superadmin ไม่สามารถเพิ่มโรงเรียนได้โดยตรง ให้ติดต่อ systemadmin (ทีมพัฒนา) เพื่อเพิ่มโรงเรียน

**Q: จะรู้ได้อย่างไรว่ามีนักเรียนเสี่ยงใหม่เข้ามา**
> A: Section B บน Dashboard refresh อัตโนมัติทุก 30 วินาที Alert ใหม่จะมี Badge กะพริบ นอกจากนี้ยังดูได้ที่ `/admin/alerts` โดยกรองสถานะ = "new"

**Q: อยากดูข้อมูลเฉพาะโรงเรียนหนึ่ง จะทำอย่างไร**
> A: ใช้ FilterBar บน Dashboard เลือก สังกัด → อำเภอ → โรงเรียน ข้อมูลทุก Section จะอัปเดตพร้อมกัน

**Q: ต้องการเปรียบเทียบผลการประเมินระหว่างอำเภอ**
> A: ที่ Section D บน Dashboard เลือกสังกัดที่ต้องการ ระบบจะ Auto-group by อำเภอ แสดง OrgCompareChart เปรียบเทียบ % ได้ทันที
